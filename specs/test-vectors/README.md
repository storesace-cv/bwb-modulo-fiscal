# Vetores fiscais

Cada diretório deve identificar o requisito e conter:

- `input.json` — intenção recebida;
- `expected.json` — resultado fiscal esperado;
- `expected-error.json` quando negativo;
- artefacto assinado/SAF-T quando aplicável;
- `README.md` com fonte, versão do pacote e explicação.

Nunca alterar silenciosamente um vetor aprovado. Criar nova versão e documentar a razão.
